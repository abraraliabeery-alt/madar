# نظام العقارات المالي المتكامل - فصل المنتجات عن العروض

## نظرة عامة

تم تطوير نظام شامل لفصل المنتجات (العقارات) عن العروض (أسعار وطرق التسويق) مع إضافة نظام محاسبي متكامل يدعم المحاسبة المزدوجة وتتبع المدفوعات والذمم.

## المشكلة التي يحلها النظام

### المشكلة السابقة:
- كان المنتج (العقار) مربوط مباشرة بسعر واحد فقط
- صعوبة في إدارة عدة أسعار لنفس العقار (للبيع والإيجار)
- عدم وجود نظام محاسبي متكامل
- صعوبة في تتبع المدفوعات والذمم
- عدم وضوح في التقارير المالية

### الحل الجديد:
- **فصل المنتج عن العرض**: نفس العقار يمكن أن يكون له عدة عروض
- **نظام محاسبي متكامل**: محاسبة مزدوجة تلقائية
- **تتبع شامل للمدفوعات**: من الطلب حتى التحصيل الكامل
- **تقارير مالية دقيقة**: لجميع الأطراف (عملاء، ملاك، مؤسسات، إدارة)

## البنية الجديدة

### 1. المنتج (Product)
```
المنتج = الأصل نفسه (الوحدة أو الأرض)
- المعلومات الأساسية: العنوان، الوصف، الموقع، الصور
- ✅ تم إزالة السعر من جدول المنتجات
- يمكن أن يكون له عدة عروض مختلفة
```

### 2. العرض (Offer)
```
العرض = طريقة التسويق أو البيع (سعر، مدة، شروط)
- أنواع العروض:
  • sale - للبيع
  • rent_monthly - إيجار شهري
  • rent_yearly - إيجار سنوي
  • rent_daily - إيجار يومي
- يحتوي على: السعر، العمولة، الشروط، فترة الصلاحية
```

### 3. العقد (Contract)
```
العقد = الربط بين المنتج والعرض والعميل والمالك
- أنواع العقود: sale (بيع), rent (إيجار)
- حالات العقد: draft, active, completed, cancelled
- يحتوي على: المبلغ الإجمالي، العربون، العمولة، الشروط
```

### 4. نظام الفواتير (Invoices)
```
الفاتورة = تفصيل المبالغ المستحقة
- أنواع الفواتير:
  • rent - فاتورة إيجار (شهرية)
  • sale - فاتورة بيع
  • deposit - فاتورة العربون
  • commission - فاتورة العمولة
  • refund - فاتورة استرداد
- حالات الفاتورة: draft, sent, paid, overdue, cancelled
```

### 5. نظام المدفوعات (Payments)
```
الدفعة = تسجيل المدفوعات للفواتير
- طرق الدفع: cash, bank_transfer, credit_card, check, online
- حالات الدفعة: pending, confirmed, failed, refunded
- تحديث تلقائي للفواتير عند التأكيد
```

### 6. نظام المحاسبة المزدوج (Accounting Entries)
```
القيد المحاسبي = تسجيل تلقائي لجميع العمليات المالية
- أنواع القيود: debit (مدين), credit (دائن)
- أنواع الحسابات:
  • revenue - الإيرادات
  • receivable - الذمم المدينة
  • commission - العمولات
  • liability - الالتزامات
  • expense - المصروفات
```

## الواجهات المطورة

### 1. واجهة العميل (Client Interface)

#### الوصول للعروض:
```http
GET /api/v1/client/offers
```
- عرض جميع العروض المتاحة مع فلاتر متقدمة
- تفاصيل كل عرض (السعر، المدة، الشروط)

#### طلب التعاقد:
```http
POST /api/v1/client/contracts
```
- طلب شراء أو إيجار عقار محدد
- التحقق من توفر العقار
- إنشاء عقد في حالة مسودة

#### إدارة العقود:
```http
GET /api/v1/client/contracts
GET /api/v1/client/contracts/{id}
DELETE /api/v1/client/contracts/{id}
```
- عرض جميع عقود العميل
- تفاصيل كل عقد مع الفواتير والمدفوعات
- إلغاء العقود في حالة المسودة

#### إدارة الفواتير والمدفوعات:
```http
GET /api/v1/client/invoices
POST /api/v1/client/payments
GET /api/v1/client/payments
```
- عرض جميع فواتير العميل
- تسجيل مدفوعات جديدة
- تتبع حالة المدفوعات

#### التقرير المالي:
```http
GET /api/v1/client/financial-summary
```
- ملخص مالي شامل للعميل
- إجمالي العقود والمدفوعات والمبالغ المتبقية

### 2. واجهة المؤسسة (Facility Interface)

#### إدارة العروض:
```http
GET /api/v1/facility/offers
POST /api/v1/facility/offers
PUT /api/v1/facility/offers/{id}
DELETE /api/v1/facility/offers/{id}
```
- إنشاء وتعديل وحذف العروض
- تفعيل/إلغاء تفعيل العروض
- نسخ العروض لمنتجات أخرى

#### إدارة العقود:
```http
GET /api/v1/facility/contracts
POST /api/v1/facility/contracts/{id}/approve
POST /api/v1/facility/contracts/{id}/reject
```
- عرض جميع العقود الخاصة بالمؤسسة
- الموافقة أو رفض طلبات التعاقد
- تتبع حالة العقود

#### إدارة المدفوعات:
```http
POST /api/v1/facility/payments/{id}/confirm
POST /api/v1/facility/payments/{id}/reject
```
- تأكيد أو رفض المدفوعات المعلقة
- تتبع جميع المدفوعات

#### التقارير المالية:
```http
GET /api/v1/facility/financial-report
GET /api/v1/facility/offers/statistics
```
- تقرير مالي شامل للمؤسسة
- إحصائيات العروض والعقود
- تحليل الأداء المالي

#### إجراءات متعددة:
```http
POST /api/v1/facility/offers/bulk-update-prices
POST /api/v1/facility/offers/{id}/copy
```
- تحديث أسعار متعددة بنسبة معينة
- نسخ العروض بين المنتجات

### 3. واجهة الأدمن (Admin Interface)

#### لوحة المعلومات المالية:
```http
GET /api/v1/admin/financial/dashboard
```
- نظرة شاملة على النظام المالي
- إحصائيات عامة وتنبيهات
- أفضل المؤسسات أداءً

#### إدارة شاملة:
```http
GET /api/v1/admin/offers
GET /api/v1/admin/contracts
GET /api/v1/admin/payments
```
- عرض وإدارة جميع العروض والعقود والمدفوعات
- تصفية وبحث متقدم
- تحديث حالة العقود

#### التقارير الشاملة:
```http
GET /api/v1/admin/financial/reports
GET /api/v1/admin/reports/customers
GET /api/v1/admin/reports/owners
GET /api/v1/admin/reports/facilities
```
- تقارير شاملة لجميع جوانب النظام
- تقارير العملاء والملاك والمؤسسات
- تحليل الأداء المالي

#### إدارة القيود المحاسبية:
```http
GET /api/v1/admin/accounting-entries
POST /api/v1/admin/accounting-entries
```
- عرض جميع القيود المحاسبية
- إنشاء قيود يدوية عند الحاجة

## سير العمل (Workflow)

### 1. إنشاء عرض جديد:
1. المؤسسة تختار منتج (عقار)
2. تحدد نوع العرض (بيع/إيجار)
3. تضع السعر والشروط
4. العرض يصبح متاح للعملاء

### 2. طلب التعاقد:
1. العميل يختار عرض معين
2. يطلب التعاقد
3. ينشأ عقد في حالة "مسودة"
4. تنشأ الفواتير تلقائياً
5. تسجل القيود المحاسبية

### 3. الموافقة على العقد:
1. المؤسسة تراجع الطلب
2. توافق أو ترفض
3. عند الموافقة: العقد يصبح "نشط"
4. العميل يتلقى إشعار

### 4. تسجيل الدفعات:
1. العميل يسجل دفعة
2. الدفعة في حالة "معلق"
3. المؤسسة تؤكد الدفعة
4. تحديث تلقائي للفواتير والقيود المحاسبية

### 5. اكتمال العقد:
1. عند سداد جميع الفواتير
2. العقد يصبح "مكتمل"
3. تحديث نهائي للقيود المحاسبية

## النظام المحاسبي

### قيود إنشاء العقد:
```
مدين: ذمم مدينة (المبلغ الإجمالي)
دائن: إيرادات (المبلغ الإجمالي)

مدين: عمولات (مبلغ العمولة)
دائن: إيرادات العمولة

دائن: التزامات (العربون - إن وجد)
```

### قيود تسجيل الدفعة:
```
مدين: نقد/بنك (مبلغ الدفعة)
دائن: ذمم مدينة (مبلغ الدفعة)
```

### قيود الإلغاء:
```
مدين: إيرادات (عكس القيد الأصلي)
دائن: ذمم مدينة (عكس القيد الأصلي)
```

## المزايا الجديدة

### للعملاء:
- **وضوح في الفواتير**: كل عقد له فواتير واضحة ومفصلة
- **تتبع المدفوعات**: معرفة المبالغ المدفوعة والمتبقية
- **مرونة في الدفع**: إمكانية الدفع على أقساط
- **شفافية كاملة**: تتبع كل خطوة في العملية

### للملاك:
- **تتبع الأرباح**: معرفة صافي المستحقات
- **تقارير مفصلة**: تقارير عن كل عقد ومستحقاته
- **شفافية العمولات**: معرفة العمولات المخصومة
- **تتبع تاريخي**: سجل كامل للعمليات المالية

### للمؤسسات:
- **مرونة في التسعير**: عدة عروض لنفس العقار
- **إدارة متقدمة**: تحكم كامل في العروض والعقود
- **تقارير مالية**: إحصائيات وتحليلات مفصلة
- **أتمتة العمليات**: تقليل الأخطاء البشرية

### للإدارة:
- **تقارير دقيقة**: إيرادات، ضرائب، عمولات، تحويلات
- **تتبع شامل**: كل المعاملات المالية
- **إدارة أفضل**: نظام محاسبي متكامل
- **شفافية كاملة**: رؤية واضحة لجميع العمليات

1. تشغيل المايجريشنز (إن لم تكن منفذة بعد)
2. نقل البيانات الموجودة من المنتجات للعروض
3. إنشاء عروض للعقود الموجودة

### المرحلة 2: التجريب
1. اختبار الواجهات الجديدة

3. مراجعة التقارير المالية
